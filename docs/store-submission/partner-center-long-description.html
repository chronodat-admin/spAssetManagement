Asset Management Hub is a SharePoint Framework (SPFx) solution for SharePoint Online that helps organizations track hardware and software assets, manage assignments and bookings, run maintenance and depreciation, and generate reports directly within Microsoft 365.

Deploy the solution once to your SharePoint Online tenant app catalog, add the web part to any modern SharePoint page or publish it as a full-page app, and use the built-in setup wizard to automatically provision the required SharePoint lists. Asset Management Hub provides dashboards, a complete asset register, assignments and returns, equipment booking and requests, barcode scanning, software license and inventory tracking, maintenance, depreciation, a report builder, and an audit log — while keeping all data securely within your Microsoft 365 tenant.

Ideal for
IT departments, managed service providers, schools and universities, healthcare and manufacturing teams, facilities and operations groups, and any organization that needs to track equipment, software licenses, and asset assignments on Microsoft 365.

Key Features

  • Complete asset register for hardware and software, with rich detail views and status tracking
  • Role-based dashboards with KPI cards and charts (totals, availability, assignments, warranty, value by location)
  • Assign and return assets to people, including bulk assign and bulk return operations
  • Book shared equipment, manage booking details, and handle asset requests with approval queues
  • Barcode / QR code scanning to quickly find and update assets
  • Software license tracking and inventory management
  • Maintenance scheduling and depreciation schedules
  • Report builder with utilization, register, depreciation, and audit reports (CSV export)
  • Warranty, license, and return-date reminders with email digests
  • Import managed devices from Microsoft Intune and bulk import assets from CSV
  • Organization setup: categories, sub-categories, vendors, locations, and projects
  • Configurable roles, permissions, forms, form templates, numbering, tags, workflows, and appearance settings
  • Audit log of create, update, and delete activity across assets, lookups, and settings

Microsoft Teams Integration
After deployment, administrators can use Sync to Teams from the SharePoint App Catalog to make the solution available in Microsoft Teams as a personal app or channel tab. Users access the same asset register, assignments, scanning, and reports directly within Teams — while SharePoint Online remains the system of record.

Email Notifications

Asset Management Hub supports flexible workflow email delivery. Choose the option that best fits your environment in Settings → Email Integration:

  • Microsoft Graph (tenant-resident, default): Notifications are sent from the signed-in user's Exchange Online mailbox. A tenant administrator approves Microsoft Graph Mail.Send once in SharePoint Admin Center → API access. Mail never leaves your Microsoft 365 tenant.
  • Chronodat Mail API (optional): Chronodat's hosted mail service sends notifications with zero tenant configuration. Requires an active Chronodat subscription. When enabled, notification recipients, subject, and message body are transmitted to the Chronodat mail service.
  • Power Automate (advanced): Not shipped with the product. You build and configure your own flows. No Mail.Send approval is required in this mode.

The app installs and runs without configuring email; only workflow notifications require a delivery option.

Microsoft Intune Sync (optional)
Administrators can import managed devices from Microsoft Intune into the asset register. This uses the Microsoft Graph DeviceManagementManagedDevices.Read.All permission, approved once by a tenant administrator in SharePoint Admin Center → API access. Intune sync is optional; the app runs fully without it.

Security & Data Residency

  • 100% tenant-resident: All asset, assignment, license, maintenance, and depreciation data is stored in standard SharePoint lists inside your Microsoft 365 tenant
  • No external data transfer by default: Default Microsoft Graph delivery keeps notification email inside your tenant
  • Microsoft 365 security inherited: Uses SharePoint permissions, Microsoft Entra ID authentication, conditional access, and DLP policies
  • Audit trail: Actions are captured through the built-in audit log, SharePoint version history, and Microsoft 365 audit logs
  • You own the data: Uninstalling leaves your SharePoint lists and data intact

Component Information (SPFx component types in this package — policy 1170.1)

  • Client-side web part — "Asset Management Hub" — for modern SharePoint pages and (via Sync to Teams) Microsoft Teams channel tabs and personal apps
  • Full-page app — the same "Asset Management Hub" web part published as a SharePoint full-page application (SharePointFullPage host) for a dedicated full-width asset management workspace
  • Form customizer extension — customizes the native new, edit, and display forms of the provisioned "AM_Assets" SharePoint list (SharePointForm host)

Prerequisites

  • Microsoft 365 with SharePoint Online (SharePoint Server / on-premises is not supported)
  • Tenant administrator permissions to deploy SPFx solutions to the app catalog
  • Modern SharePoint sites
  • Microsoft Exchange Online mailboxes (for Microsoft Graph workflow email notifications)
  • Microsoft Intune (optional) for managed-device import
  • Microsoft Teams (optional) for Teams integration

Limitations & Out of Scope

  • SharePoint Online only — not supported on SharePoint Server or on-premises farms
  • One license per site collection — data and configuration are scoped to the licensed site collection
  • Setup wizard required — a site owner or administrator must run first-time list provisioning before most features are available
  • No native mobile app — runs in supported desktop browsers and Microsoft Teams; barcode scanning uses the device browser camera or manual code entry
  • No built-in ERP or CMDB integration — asset data is exported via CSV; you connect downstream systems yourself
  • Workflow rules and scheduled reports — require Power Automate or a backend runner to execute; the app stores the configuration
  • Role-based data visibility — members see scoped records; managers and administrators see broader or organization-wide data

Recommended Browsers

  • Microsoft Edge
  • Google Chrome
